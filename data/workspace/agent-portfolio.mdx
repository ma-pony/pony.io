## 作品集目标

作品集不是兴趣项目，也不是创业 MVP，而是求职证据。

每个项目都要证明：

- 老板能把 Agent 技术接到真实业务流程里。
- 老板懂 RAG、tool calling、workflow、eval、human-in-the-loop。
- 老板能把技术方案翻译成业务价值。

## 作品集总标准

每个项目一页，包含：

- 项目名。
- 目标岗位。
- 业务场景。
- 用户 / 使用者。
- 痛点。
- Agent 流程图。
- 技术栈。
- 核心代码或核心逻辑说明。
- 评测指标。
- 失败案例。
- 业务价值。
- 简历 bullet。
- 面试讲解稿。
- 可用于触达的一句话。

## 优先级

1. 岗位机会分析 Agent。
2. Self-Improving Research Agent。
3. 企业知识库 + 销售问答 Agent。
4. 内容运营 Agent。

## 项目 1：岗位机会分析 Agent

目标：扫描岗位、抽取要求、分析能力缺口、生成作品集建议和触达话术。

对应岗位：AI Agent 应用工程、自动化工作流、个人效率工具、职业服务场景。

MVP 功能：

- 输入岗位链接或岗位文本。
- 抽取公司、岗位、要求、技术栈、业务场景、经验要求。
- 匹配老板现有能力。
- 输出能力缺口。
- 生成作品集建议。
- 生成触达话术。
- 保存到 Obsidian 机会清单。

## 项目 2：Self-Improving Research Agent

目标：持续监控 AI Agent / LLM / RAG / Eval 方向论文、GitHub 项目和行业信号，生成证据化研究简报与社媒草稿，并根据人工反馈逐步优化筛选和写作策略。

对应岗位：AI Agent 应用工程、RAG 工程、Agent Workflow、AI 自动化、研究工具方向。

MVP 功能：

- 采集 GitHub / 论文 / 博客 / 行业信号。
- 判断候选内容适合日报、深度拆解、作品集还是社媒草稿。
- 抽取 README / 论文 / 核心代码证据。
- 生成日报段落、研究笔记和 X 草稿。
- 记录人工反馈。
- 把反馈转成 memory / rubric 更新建议。
- 用 golden set 做回归评测，避免越改越差。

可展示点：

- 长期记忆：episodic / semantic / procedural / eval memory。
- 自我改进闭环：feedback -> memory -> eval -> policy update。
- 评测体系：relevance、evidence quality、actionability、novelty、factuality、format stability。
- human-in-the-loop：公开发布和策略更新都保留人工审核。

详细计划：[Self-Improving Research Agent 作品集实施计划](/workspace/self-improving-research-agent-plan)。

## 项目 3：企业知识库 + 销售问答 Agent

目标：做一个企业知识库场景的 RAG Agent，用于销售或客服问答。

对应岗位：RAG 应用开发、AI Agent 应用工程、企业 AI 解决方案。

MVP 功能：

- 文档导入。
- 文档 chunk。
- embedding 检索。
- hybrid search / rerank。
- 回答生成。
- 引用来源。
- 反馈记录。
- 简单 eval。

## 项目 4：内容运营 Agent

目标：把已有日报、博客、社媒流程整理成一个可展示的内容运营 Agent 案例。

对应岗位：AI 自动化、增长工程、Agent 产品落地、内容运营 AI 工具。

MVP 功能：

- 采集 GitHub / 新闻 / 博客 / 社媒信号。
- 筛选 AI Agent 相关内容。
- 生成日报。
- 改写成 X/Twitter 内容。
- 保留人工审核。
- 发布到博客或社媒。
- 记录反馈。

## 下一步

当前排序：岗位机会分析 Agent 仍是最直接服务求职的 P0；Self-Improving Research Agent 作为第二主线，负责沉淀更强的 memory / eval / 自我改进能力，并反哺日报、X 和博客内容。
