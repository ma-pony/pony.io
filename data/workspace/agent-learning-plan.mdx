## 0. 核心定位

老板当前没有 AI Agent 相关正式工作经验，所以这个求职计划的核心不是先包装经验，而是先补出一条能被招聘方相信的能力链：

> 学理论 -> 做小实验 -> 写学习笔记 -> 做作品集 -> 形成面试表达 -> 再去触达机会。

学习计划不是泛泛学 AI，也不是为了读论文而读论文。它只服务一个目标：

> 让老板在 AI Agent 应用工程 / RAG 应用开发 / 大模型产品落地方向，具备可解释、可演示、可面试的基础能力。

硬约束：

- 每个学习点都必须转化为一个输出：笔记、实验、作品集功能、面试回答、简历 bullet 或触达材料。
- 不追求一开始就很深，但要形成闭环：能讲清楚、能动手做、能知道边界。
- 不和算法博士拼论文和模型训练，重点补工程型理论和落地判断。

## 1. 学习目标

### 1.1 30 天目标

30 天后要达到：

- 能讲清楚 AI Agent、RAG、tool calling、memory、eval 的基本原理和常见失败模式。
- 能做出 1 个可演示的 AI Agent / RAG 小作品。
- 能把学习内容转化成 10-15 个面试问题回答。
- 能写出一页 AI Agent 方向简历项目经历草稿。
- 能对一个 AI Agent 岗位 JD 做能力差距分析。

### 1.2 60 天目标

60 天后要达到：

- 完成 2 个作品集项目初版。
- 能展示 RAG eval、tool calling、human-in-the-loop、失败处理和成本意识。
- 能围绕目标岗位进行主动触达。
- 能进行基础面试，不再停留在“没有相关经验”。

### 1.3 90 天目标

90 天后要达到：

- 形成稳定作品集页面。
- 形成 AI Agent 学习笔记体系。
- 形成 20-30 个面试回答。
- 开始拿到面试、约聊、内推或项目合作反馈。

## 2. 学习路径总览

学习分 6 层，按求职优先级排序：

1. **LLM 应用基础**：知道模型能做什么、不能做什么。
2. **RAG 生产化基础**：企业 Agent 最常见入口。
3. **Agent Workflow**：从“调 API”升级到“做任务系统”。
4. **Tool Calling / Browser Automation**：让 agent 能操作外部工具。
5. **Eval / 评测体系**：证明系统有效，体现生产化能力。
6. **Memory / Context / 产品经济学**：形成差异化判断和面试加分项。

路线原则：

- 先能做，再做深。
- 先学岗位高频要求，再学前沿概念。
- 每周一个主题，每天一个小输出。
- 理论和实验绑定，避免只会背概念。

## 3. 30 天学习路线图

### 第 1 周：LLM 应用基础 + Prompt / API / 结构化输出

目标：建立最小共同语言，知道大模型应用的基本构件。

要学会：

- token、context window、temperature、top-p。
- prompt、system prompt、few-shot、chain-of-thought 的使用边界。
- JSON / schema structured output。
- function calling / tool calling 的基本概念。
- 模型幻觉是什么，为什么会发生。
- 为什么“会调 API”不等于“会做 Agent”。

每日安排：

#### Day 1：LLM 应用全景

输出：一页笔记《LLM 应用和 AI Agent 的区别》。

要回答：

- Chatbot、workflow、agent 的区别是什么？
- 什么时候不需要 agent？
- 为什么岗位会强调 RAG、tool calling、eval？

#### Day 2：Prompt 和结构化输出

输出：一个小实验：让模型把岗位 JD 抽成 JSON。

实验内容：

- 输入一段岗位描述。
- 输出字段：岗位名、技术栈、业务场景、经验要求、加分项。
- 对比普通自然语言输出和 JSON schema 输出的稳定性。

#### Day 3：幻觉和可靠性

输出：一段面试回答《为什么大模型会幻觉，工程上怎么降低风险》。

要点：

- 模型生成的是概率分布，不是真实数据库查询。
- 缺少上下文、检索错误、prompt 模糊都会导致幻觉。
- 工程手段：RAG、引用来源、结构化输出、人工审核、eval。

#### Day 4：Tool Calling 入门

输出：一个小实验：让模型判断何时调用搜索 / 读取文件 / 计算工具。

要点：

- tool calling 的本质是让模型选择工具和参数。
- 失败模式：选错工具、参数错、工具结果读错、循环调用。

#### Day 5：Mini 项目 1 - JD 解析器

输出：一个最小岗位 JD 解析器设计文档。

功能：

- 输入岗位文本。
- 输出结构化要求。
- 标记能力缺口。
- 生成学习建议。

#### Day 6：复盘和面试题

输出：补 3 个面试题回答到 [AI Agent 面试题库](/workspace/agent-interview-bank)。

建议题目：

- 为什么只会 prompt 不够？
- 什么是结构化输出？
- tool calling 和普通 API 调用有什么区别？

#### Day 7：周总结

输出：一篇周学习总结，说明本周学到的东西如何服务求职。

验收标准：

- 能讲清楚 chatbot、workflow、agent 的区别。
- 能把岗位 JD 抽成结构化字段。
- 能说出 3 个 LLM 应用风险和对应工程手段。

### 第 2 周：RAG 生产化基础

目标：掌握企业 AI Agent 最常见的能力入口。

要学会：

- RAG 是什么，解决什么问题。
- chunk size / overlap。
- embedding 和向量检索。
- BM25、hybrid search、rerank。
- citation grounding。
- RAG eval 的基本指标。

每日安排：

#### Day 8：RAG 全景

输出：一页笔记《RAG 为什么是企业 Agent 的入口》。

要回答：

- 为什么不直接把文档塞进长上下文？
- RAG 解决什么问题，不解决什么问题？
- 企业为什么需要引用来源？

#### Day 9：Chunk 和检索

输出：小实验设计：同一文档用不同 chunk size 检索同一个问题。

要观察：

- chunk 太小会丢上下文。
- chunk 太大召回不精确。
- overlap 能缓解断裂，但增加成本。

#### Day 10：Embedding / BM25 / Hybrid Search

输出：面试回答《BM25、embedding、hybrid search 的区别》。

要点：

- BM25 擅长关键词精确匹配。
- embedding 擅长语义相似。
- hybrid search 结合两者。
- rerank 负责最后排序优化。

#### Day 11：Rerank 和引用来源

输出：一个 RAG 回答格式模板，必须带引用来源和“不确定”处理。

模板字段：

- 回答。
- 引用来源。
- 置信度。
- 不确定点。
- 建议人工确认项。

#### Day 12：RAG Eval 入门

输出：10 条 golden set 样例。

字段：

- 问题。
- 标准答案。
- 应检索到的文档片段。
- 允许答案范围。
- 错误类型。

#### Day 13：Mini 项目 2 - 知识库问答设计

输出：企业知识库 + 销售问答 Agent 的一页方案。

必须包含：

- 数据来源。
- RAG 流程。
- eval 指标。
- 人工审核。
- 业务价值。

#### Day 14：周总结

输出：补 4 个 RAG 面试题到 [AI Agent 面试题库](/workspace/agent-interview-bank)。

验收标准：

- 能讲清楚 RAG 流程。
- 能解释 chunk、embedding、BM25、rerank。
- 能设计最小 RAG eval。

### 第 3 周：Agent Workflow + Tool Calling

目标：从“问答系统”升级到“任务系统”。

要学会：

- ReAct。
- Plan-and-Execute。
- planner / executor / critic。
- tool calling 失败模式。
- browser automation。
- human-in-the-loop。
- agent loop 退出条件。

每日安排：

#### Day 15：Agent 架构全景

输出：一页笔记《Agent、Workflow、RAG 的关系》。

要回答：

- 什么任务适合 agent？
- 什么任务普通 workflow 更好？
- Agent 的不稳定性来自哪里？

#### Day 16：ReAct 和 Plan-and-Execute

输出：面试回答《ReAct 和 Plan-and-Execute 有什么区别》。

要点：

- ReAct 边想边做，适合探索型任务。
- Plan-and-Execute 先规划再执行，适合步骤清晰任务。
- 复杂任务需要检查点和退出条件。

#### Day 17：Tool Calling 失败模式

输出：失败模式清单。

至少包含：

- 工具选错。
- 参数错。
- 工具结果为空。
- 工具结果过长。
- 模型误读工具结果。
- 无限循环。
- 权限和安全问题。

#### Day 18：Human-in-the-loop

输出：一页笔记《Agent 哪里必须保留人工审核》。

重点场景：

- 发帖。
- 发邮件 / 私信。
- 删除或修改数据。
- 交易 / 投资。
- 对外承诺。

#### Day 19：Mini 项目 3 - 岗位机会分析 Agent 方案

输出：岗位机会分析 Agent 的详细方案。

流程：

- 采集岗位。
- 抽取要求。
- 聚类关键词。
- 匹配老板能力。
- 生成学习建议。
- 生成触达话术。
- 人工确认。

#### Day 20：Agent Eval

输出：给岗位机会分析 Agent 设计 5 个 eval case。

评测点：

- 抽取准确率。
- 缺口分析是否合理。
- 触达话术是否贴合岗位。
- 是否编造不存在的岗位要求。

#### Day 21：周总结

输出：补 4 个 Agent workflow 面试题到 [AI Agent 面试题库](/workspace/agent-interview-bank)。

验收标准：

- 能讲清楚 agent workflow 基本结构。
- 能说出 tool calling 失败模式和兜底方式。
- 能设计一个简单的 agent eval。

### 第 4 周：Eval + Memory + 产品经济学 + 求职表达

目标：把学习转化成面试材料和作品集表达。

要学会：

- eval 为什么重要。
- LLM-as-judge 的偏差。
- memory 和 context 的区别。
- cost / latency / quality tradeoff。
- 如何判断一个任务值不值得 agent 化。
- 如何讲自己的项目。

每日安排：

#### Day 22：Eval 系统化

输出：一页笔记《没有 eval 就没有可持续优化》。

要回答：

- 为什么 demo 成功不等于生产可用？
- golden set 是什么？
- case replay 怎么用？
- 人工评分和 LLM-as-judge 如何结合？

#### Day 23：LLM-as-judge

输出：面试回答《LLM-as-judge 有什么偏差，怎么控制》。

要点：

- 偏好长答案。
- 容易被格式影响。
- 对事实错误不一定敏感。
- 需要人工抽检和标准答案。

#### Day 24：Memory 和 Context

输出：面试回答《为什么长上下文不等于记忆》。

要点：

- context 是当前输入窗口。
- memory 需要写入、检索、更新、遗忘。
- 记忆要有作用域、权限和过期机制。

#### Day 25：AI 产品经济学

输出：一页判断框架《什么任务值得 agent 化》。

判断维度：

- 任务价值。
- 重复频率。
- 容错率。
- 人工成本。
- 数据可得性。
- 工具可操作性。
- 失败风险。
- 评测难度。

#### Day 26：项目讲解稿

输出：岗位机会分析 Agent 或 RAG Agent 的 2 分钟项目讲解稿。

结构：

- 背景。
- 问题。
- 方案。
- 技术栈。
- eval。
- 失败处理。
- 业务价值。

#### Day 27：简历转化

输出：把学习和小实验转成 3 条简历 bullet。

原则：

- 不夸大为工作经验。
- 写成项目 / 学习项目 / 自主构建。
- 强调闭环和可验证输出。

#### Day 28：模拟面试

输出：10 个高频问题的简答版。

题目来自 [AI Agent 面试题库](/workspace/agent-interview-bank)。

#### Day 29：作品集整理

输出：作品集页面 V1。

至少包含：

- 一个项目方案。
- 一个小实验。

## 6. 学习和作品集绑定关系

| 学习模块 | 对应作品集 | 求职价值 |
|---|---|---|
| LLM 结构化输出 | 岗位机会分析 Agent | 展示信息抽取和结构化能力 |
| RAG | 企业知识库 + 销售问答 Agent | 对应岗位最多，最容易被招聘方理解 |

| AI 产品经济学 | 所有项目 | 展示业务落地判断 |

## 7. 当前阶段最重要的事

因为老板现在没有相关工作经验，所以第一阶段不要急着大量触达。优先顺序应该是：

1. 学出基本概念和表达。
2. 做出小实验。
3. 把小实验串成作品集雏形。
4. 把作品集转成简历和面试话术。
5. 再开始更主动地触达岗位和团队。

但也不能完全不看岗位。每天仍然要看少量 JD，用来校准学习方向。

## 8. 第一周立即执行清单

1. 建立第一篇学习笔记：LLM 应用和 AI Agent 的区别。
2. 找 3 个 AI Agent / RAG 岗位 JD，抽取共同要求。
3. 做一个 JD -> JSON 的结构化输出小实验。
4. 写第一版面试回答：为什么只会调用 API 不等于会做 AI Agent。
5. 更新 [AI Agent 面试题库](/workspace/agent-interview-bank) 的前 3 个问题。

## 9. 一句话总结

> 当前最重要的不是包装已有经验，而是用 30 天把 AI Agent 求职所需的基础能力学出来、做出来、写出来、讲出来。
