## 0. 北极星目标

这个计划只有一个核心目标：

> 帮老板找到 AI Agent 相关的工作机会。

这里的“工作机会”包括：

- AI Agent 应用工程岗位。
- RAG / 大模型应用开发岗位。
- AI Agent 产品 / 解决方案岗位。
- AI 自动化 / 增长工程岗位。
- AI Agent 项目制、顾问、合作机会。

硬约束：

> 所有动作都必须回答：这件事是否提高老板拿到 AI Agent 相关工作机会的概率？

如果不能提高，就降级、推迟或删除。

这个计划不是泛内容运营计划，不是泛创业计划，不是泛学习计划，也不是为了把 agent 系统做得更复杂。agent 系统只是工具，目标是找到相关工作机会。

## 1. 战略定位

老板不应该包装成“只会接 API 的 AI 应用开发者”，也不应该包装成“纯算法研究员”。更适合的定位是：

> AI Agent 应用工程 / 产品落地 / 自动化增长方向的人，能把 Agent 技术接到真实业务流程里，并用结果证明价值。

这个定位的优势是：

- 老板有全栈能力，能快速做 demo 和交付。
- 老板懂内容、网站、自动化、社媒和运营，有真实业务场景感。
- 老板关注赚钱和结果，适合产品落地、解决方案、增长和顾问型机会。
- 老板正在运行自己的 agent / 日报 / 社媒 / 自动化系统，这本身可以变成作品集。

一句话职业标签：

> AI Agent 应用工程 / RAG 应用开发 / 大模型产品落地

一句话自我定位：

> 我擅长把 AI Agent、RAG、工具调用和自动化工作流接入真实业务流程，用可评估的系统帮助团队省人、提效、提升转化或增强决策。

## 2. 目标岗位画像

### 2.1 优先目标岗位

#### A. AI Agent 应用工程师

关键词：Agent workflow、RAG、tool calling、browser automation、LLM API、Python、业务系统集成、任务队列、错误恢复。

老板卖点：

- 能把业务流程拆成 agent workflow。
- 能做数据接入、工具调用、浏览器自动化、定时任务和发布流程。
- 能讲清楚失败处理、人工审核、成本和延迟。

必须准备的证据：

- 一个真实 agent workflow 案例。
- 架构图或流程图。
- 失败案例和改进记录。
- 评测或效果指标。

#### B. RAG / 大模型应用开发工程师

关键词：知识库、企业文档、embedding、hybrid search、rerank、citation、权限、RAG eval、LLM API。

老板卖点：

- 能做生产化 RAG，而不是只做聊天 demo。
- 能解释 chunk、召回、rerank、引用、权限、评测的取舍。
- 能把 RAG 放进销售、客服、知识库、内部提效等场景。

必须准备的证据：

- 企业知识库 + 销售问答 Agent。
- RAG eval 样例。
- 引用来源和错误处理。
- 一篇案例说明。

#### C. AI Agent 产品经理 / 解决方案顾问

关键词：需求分析、PRD、客户调研、交付、业务指标、转化率、降本增效、AI SaaS。

老板卖点：

- 懂技术，也懂业务结果。
- 能把客户场景翻译成可落地的 agent 方案。
- 能定义指标，判断 agent 值不值得做。

必须准备的证据：

- 一个解决方案文档。
- 一个业务流程 agent 化拆解。
- 一个 ROI / 成本收益分析样例。
- 触达话术和 demo 组合。

#### D. AI 自动化 / 增长工程方向

关键词：内容自动化、销售线索、运营流程、数据采集、社媒增长、自动化工作流、CRM。

老板卖点：

- 已有日报、博客、社媒、内容分发、自动化采集的真实经验。
- 能把 agent 用在内容、销售、运营、增长流程中。
- 适合出海营销、销售 Agent、内容工具、AI SaaS 类机会。

必须准备的证据：

- 内容运营 Agent 案例。
- 发布流程和人工审核节点。
- 节省时间或提升质量的指标。
- 公开内容背书。

### 2.2 不优先主攻

- 纯算法研究岗。
- 纯模型训练岗。
- 要求强论文、强数学、强实验室背景的岗位。
- 和业务落地弱相关的泛 AI 内容岗位。

这些方向可以补基本概念，用来面试沟通和判断边界，但不作为主战场。

## 3. 总工作流

所有 agent 工作都围绕求职机会推进：

```text
岗位扫描 -> 能力差距分析 -> 学习补齐 -> 作品集证明 -> 内容背书 -> 主动触达 -> 面试复盘 -> 调整定位
```

### 3.1 岗位扫描

目标：知道市场真正要什么，不凭想象学习。

输入：

- BOSS 直聘、猎聘、拉勾、LinkedIn、脉脉、X、GitHub、公司官网招聘页。
- AI SaaS、RAG、企业知识库、智能客服、出海营销、销售 Agent、内容自动化公司。

输出：

- 目标岗位清单。
- 高频关键词。
- 能力缺口。
- 可触达对象。

### 3.2 能力差距分析

目标：把岗位要求翻译成老板需要补的具体能力。

输出格式：

```text
岗位要求：熟悉 RAG 评测
老板现状：知道 RAG 基础，但缺少可展示 eval 案例
补齐动作：做 20 条 golden set + 3 个失败案例 + 一页评测说明
可展示证据：作品集项目中的 RAG eval section
```

### 3.3 学习补齐

目标：只学能提高求职胜率的内容。

学习必须对应至少一个用途：

- 面试能讲。
- 项目能用。
- 简历能写。
- 触达能说。
- 作品集能展示。

### 3.4 作品集证明

目标：让招聘方相信老板真的能做，不只是会说。

每个作品集都要回答：

- 场景是什么？
- 谁会用？
- 原流程痛点是什么？
- Agent 怎么拆流程？
- 数据怎么进？
- 工具怎么调？
- 失败怎么处理？
- 怎么评测？
- 成本和延迟如何控制？
- 业务价值是什么？

### 3.5 内容背书

目标：为求职和触达提供公开可信材料。

内容不是为了流量，而是为了让别人看到：老板持续理解 AI Agent 落地，能讲清楚工程和业务。

### 3.6 主动触达

目标：把计划真正推向机会。

触达优先级高于泛内容和泛学习。没有触达，就没有真实市场反馈。

### 3.7 面试复盘

目标：把每次反馈转化为下一轮胜率提升。

复盘内容：

- 被问了什么？
- 哪些回答不够硬？
- 哪些项目证据不够？
- 哪些岗位其实不匹配？
- 下一步补什么？

## 4. 现有 agent / cron 的重新定位

### 4.1 科技 / AI 日报

现有任务：`tech-ai-daily-brief`，每天 9:00。

定位：服务岗位扫描、面试谈资和作品集选题。

日报不是泛科技资讯，也不是为了做媒体号。它要服务三件事：

1. 发现 AI Agent 相关岗位和商业场景的新趋势。
2. 提炼面试、触达、作品集可用的谈资。
3. 找到值得做成 demo 的开源项目或业务流程。

筛选标准：

- 是否和目标岗位关键词相关：Agent、RAG、tool calling、memory、eval、workflow、browser automation、AI SaaS、enterprise AI。
- 是否能帮助判断岗位市场变化。
- 是否能变成一个作品集项目。
- 是否能变成面试中的高质量案例。
- 是否能辅助主动触达某类公司。

日报新增输出字段：

- **岗位相关性**：对应哪类岗位要求。
- **能力提示**：老板需要补什么能力。
- **作品集机会**：是否值得做 demo。
- **面试谈资**：能否变成一句专业判断。
- **触达线索**：是否出现值得联系的公司、项目或作者。

### 4.2 X/Twitter 账号维护

现有任务：`twitter-account-maintainer`，每天 9:30。

定位：职业背书和机会触达工具。

推特不是泛流量运营，而是让招聘方、创业者、同行看到老板持续关注 AI Agent 落地。

内容优先级：

1. AI Agent 落地判断。
2. RAG / Agent 工程实践笔记。
3. 开源项目拆解，尤其是核心代码逻辑和可用场景。
4. 作品集进展和案例复盘。
5. 岗位市场观察。

每条内容都要尽量能服务一个求职用途：

- 可放进作品集页。
- 可附在触达消息里。
- 可作为面试谈资。
- 可展示老板的判断能力。

不优先：

- 追热点式泛 AI 评论。
- 只求互动、不服务职业定位的内容。
- 与 AI Agent 工作机会无关的泛运营内容。

### 4.3 作品集推进

现有任务：`ai-agent-daily-portfolio-builder`，每天 20:30。

定位：求职证据生产器。

作品集不是创业 MVP，也不是纯兴趣项目，而是简历、触达和面试的证据。

优先级：

1. 岗位机会分析 Agent。
2. 企业知识库 + 销售问答 Agent。
3. 内容运营 Agent。

排序理由：

- 岗位机会分析 Agent 与当前目标最一致，可以直接服务求职。
- 企业知识库 + 销售问答 Agent 对应市场最大公约数 RAG 需求。
- 内容运营 Agent 利用老板现有系统，容易形成真实案例。

### 4.4 职业机会扫描

现有任务：`ai-agent-career-coach`，每周一、周四 9:00。

定位：目标岗位和策略更新。

输出必须包含：

- 新增岗位 / 公司。
- 高频岗位要求。
- 老板匹配点。
- 当前缺口。
- 本周最该补的作品集或材料。

### 4.5 机会触达

现有任务：`ai-agent-daily-outreach-scout`，工作日 16:00。

定位：核心转化动作。

这是整个计划里最接近结果的动作，优先级高于泛内容和泛学习。

输出必须包含：

- 1-3 个可触达对象。
- 为什么值得触达。
- 用什么材料触达。
- 触达话术草稿。
- 下一步动作。

### 4.6 理论学习

现有任务：`ai-agent-daily-theory-drill`，每天 8:00。

定位：求职能力补齐。

学习不是为了知识收藏，而是为了：

1. 看懂岗位要求。
2. 面试能讲清楚。
3. 项目能做出来。
4. 方案能判断优劣。

每日学习输出必须至少符合一种：

- 一段面试可用解释。
- 一个作品集改进点。
- 一条职业背书内容。
- 一个小实验记录。
- 一个岗位要求翻译成能力清单。

## 5. 作品集计划

### 5.1 作品集总标准

每个作品集项目都必须有：

- 项目名。
- 目标岗位。
- 业务场景。
- 用户 / 使用者。
- 痛点。
- Agent 流程图。
- 技术栈。
- 核心代码或核心逻辑说明。
- 评测指标。
- 失败案例。
- 业务价值。
- 简历 bullet。
- 面试讲解稿。
- 可用于触达的一句话。

### 5.2 作品 1：岗位机会分析 Agent

目标：做一个直接服务求职的 agent，用来扫描岗位、抽取要求、分析能力缺口、生成触达动作。

对应岗位：AI Agent 应用工程、自动化工作流、个人效率工具、职业服务场景。

为什么优先：

- 和当前目标完全一致。
- 能展示采集、结构化抽取、分析、推荐、生成话术的完整 agent workflow。
- 做出来后可以反过来推进求职。

MVP 功能：

1. 输入岗位链接或岗位文本。
2. 抽取公司、岗位、要求、技术栈、业务场景、经验要求。
3. 匹配老板现有能力。
4. 输出能力缺口。
5. 生成作品集建议。
6. 生成触达话术。
7. 保存到 Obsidian 机会清单。

可展示点：

- 信息抽取 schema。
- 岗位要求聚类。
- 能力差距分析。
- 自动生成触达话术。
- 人工审核节点。

简历 bullet 草稿：

> 构建 AI Agent 岗位机会分析工作流，自动抽取岗位要求、聚类能力关键词、分析个人匹配度并生成触达话术，用于持续优化 AI Agent 方向求职策略。

面试讲解重点：

- 为什么岗位信息适合结构化抽取。
- 如何避免 LLM 编造岗位要求。
- 如何把能力匹配做成可解释，而不是黑箱打分。
- 如何保留人工审核，避免错误触达。

### 5.3 作品 2：企业知识库 + 销售问答 Agent

目标：做一个企业知识库场景的 RAG Agent，用于销售或客服问答。

对应岗位：RAG 应用开发、AI Agent 应用工程、企业 AI 解决方案。

MVP 功能：

1. 上传或导入文档。
2. 文档 chunk。
3. embedding 检索。
4. 可选 BM25 / hybrid search。
5. rerank。
6. 回答生成。
7. 引用来源。
8. 反馈记录。
9. 简单 eval。

可展示点：

- chunk 策略对召回的影响。
- embedding / BM25 / rerank 的取舍。
- 引用来源和防幻觉。
- golden set 评测。
- 错误案例分析。

简历 bullet 草稿：

> 设计并实现企业知识库 RAG 问答 Agent，支持文档切分、向量检索、引用溯源、回答质量评测和反馈记录，面向销售 / 客服场景提升知识查询效率。

面试讲解重点：

- 为什么 RAG 是企业 Agent 的入口。
- 如何评估 answer faithfulness 和 context recall。
- 为什么长上下文不能替代检索。
- 如何处理权限、引用和错误答案。

### 5.4 作品 3：内容运营 Agent

目标：把已有日报、博客、社媒流程整理成一个可展示的内容运营 Agent 案例。

对应岗位：AI 自动化、增长工程、Agent 产品落地、内容运营 AI 工具。

MVP 功能：

1. 采集 GitHub / 新闻 / 博客 / 社媒信号。
2. 筛选 AI Agent 相关内容。
3. 生成日报。
4. 改写成 X/Twitter 内容。
5. 保留人工审核。
6. 发布到博客或社媒。
7. 记录反馈。

可展示点：

- 多源采集。
- 内容筛选标准。
- 人工审核节点。
- 发布链路。
- 从日报到社媒再到作品集选题的转化。

简历 bullet 草稿：

> 搭建 AI 内容运营 Agent 工作流，自动采集多源技术信号、生成日报、改写社媒内容并发布到博客 / X，保留人工审核与反馈记录，用于提升内容生产效率和职业背书。

面试讲解重点：

- Agent 不是全自动发布，而是 human-in-the-loop。
- 如何控制内容质量和事实错误。
- 如何把信息流变成职业机会线索。
- 如何衡量节省时间和内容质量。

## 6. 学习计划

### 6.1 学习优先级

学习只服务求职胜率。优先顺序：

1. RAG 生产化。
2. Agent 工作流。
3. 评测体系。
4. 记忆和上下文工程。
5. AI 产品经济学。
6. LLM 基础和对齐基本概念。

### 6.2 模块 1：RAG 生产化

必须掌握：

- chunk size / overlap。
- embedding。
- BM25。
- hybrid search。
- rerank。
- query rewrite。
- citation grounding。
- context precision / recall。
- faithfulness。
- 权限和数据边界。

面试目标：

> 能讲清楚为什么 RAG 会漏召回、为什么会幻觉、如何用 eval 找问题、如何用 rerank 和 query rewrite 改进。

项目绑定：企业知识库 + 销售问答 Agent。

### 6.3 模块 2：Agent 工作流

必须掌握：

- ReAct。
- Plan-and-Execute。
- tool calling。
- planner / executor / critic。
- browser automation。
- 错误恢复。
- human-in-the-loop。
- agent loop 退出条件。

面试目标：

> 能讲清楚什么时候需要 agent，什么时候普通 workflow 更好；能说出 tool calling 的失败模式和兜底方式。

项目绑定：岗位机会分析 Agent、内容运营 Agent。

### 6.4 模块 3：评测体系

必须掌握：

- golden set。
- case replay。
- task success rate。
- tool success rate。
- LLM-as-judge 偏差。
- 人工评分。
- 成本、延迟、质量三角。

面试目标：

> 能讲清楚“没有 eval 就没有可持续优化”，并能展示自己项目里的评测表。

项目绑定：三个作品集都必须有 eval section。

### 6.5 模块 4：记忆和上下文工程

必须掌握：

- short-term memory。
- long-term memory。
- episodic memory。
- semantic memory。
- memory retrieval。
- context compression。
- 用户画像和隐私边界。

面试目标：

> 能讲清楚长上下文不是记忆，记忆系统需要检索、压缩、更新和遗忘机制。

项目绑定：岗位机会分析 Agent 的历史反馈、内容运营 Agent 的选题池。

### 6.6 模块 5：AI 产品经济学

必须掌握：

- inference cost。
- latency / quality / cost tradeoff。
- 开源模型 vs API 模型。
- 自动化 ROI。
- 什么任务适合 agent。

## 10. 每日最小动作

每天只保留和工作机会强相关的动作：

1. 看 3-5 条 AI Agent / RAG / 大模型应用岗位或行业信号。
2. 记录 1 条岗位要求或市场变化。
3. 推进 1 个作品集小动作。
4. 学 1 个能用于面试或项目的概念。
5. 发布或沉淀 1 条职业背书内容。

## 13. 衡量指标

这个计划是否有效，不看发了多少内容、自动化了多少流程，而看：

### 过程指标

- 每周新增机会数量。
- 每周触达数量。
- 每周简历 / 话术 / 作品集更新次数。
- 每周学习是否转化为项目或面试材料。

### 结果指标

- 是否形成清晰目标岗位清单。
- 是否持续发现真实工作机会。
- 是否完成可用于求职的作品集。
- 是否拿到面试、约聊、内推、合作反馈。
- 是否能更准确回答岗位要求。
- 是否能在面试中讲清楚 Agent 系统设计和业务价值。
- 是否最终转化成 AI Agent 相关工作机会。

## 14. 风险和纠偏规则

### 风险 1：偏成泛内容运营

表现：每天发内容，但没有岗位、触达、作品集、面试材料。

纠偏：内容必须绑定一个用途：触达、作品集、简历、面试谈资。

### 风险 2：偏成泛学习

表现：学很多理论，但没有 demo、简历和面试表达。

纠偏：每个学习点都要输出面试解释、项目改进或作品集材料。

### 风险 3：偏成泛创业

表现：开始想产品商业化，但忘了当前目标是找工作机会。

纠偏：创业想法只保留为作品集和触达话题，除非它直接带来岗位、合作或项目制机会。

### 风险 4：只做系统不触达

表现：一直优化 agent 系统，但没有联系公司和人。

纠偏：每周至少 5 次轻触达，没有触达就视为计划执行失败。

### 风险 5：目标岗位太散

表现：一会儿算法岗，一会儿产品岗，一会儿运营岗。

纠偏：主线锁定 AI Agent 应用工程 / RAG 应用开发 / 大模型产品落地，其他只作为补充。

## 16. 一句话总结

> 这个计划的本质不是做一个泛 Agent 系统，而是用 agent 系统反向推进老板找到 AI Agent 相关工作机会：扫描岗位、补齐能力、制作作品集、建立职业背书、主动触达，并通过反馈不断提高命中率。
