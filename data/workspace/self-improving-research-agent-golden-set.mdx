## 0. 目的

这份 golden set 用来验证 Self-Improving Research Agent 的策略更新是否真的提升质量，而不是只在新样本上看起来更好。

每次修改以下内容后，至少抽 5 条样本回归：

- source policy
- ranking policy
- prompt
- writing rubric
- memory 写入规则
- feedback 抽取规则

## 1. 评分维度

每项 1-5 分。

| 指标 | 说明 |
|---|---|
| relevance | 是否服务 AI Agent 学习 / 求职 / 作品集主线 |
| evidence_quality | 是否有原文、代码、论文或数据证据 |
| actionability | 是否给出具体动作，而不只是概念判断 |
| novelty | 是否避免重复旧内容，是否有新信息增量 |
| factuality | 是否事实准确，是否区分证据和推测 |
| format_stability | 是否符合目标输出格式 |
| portfolio_value | 是否能转成作品集、面试表达或公开内容 |

额外记录：

- cost_minutes
- token_cost_estimate
- human_decision：adopt / revise / reject
- failure_tags

## 3. 初始样本

### GS-001：持续学习 Survey

```yaml
id: GS-001
source_type: paper
input: https://victorchen96.github.io/continual_learning_survey.pdf
expected_output_type: deep_dive
must_include:
  - three-axis taxonomy: what / how / when
  - stability-plasticity dilemma
  - catastrophic forgetting
  - self-improvement collapse
  - 和 Agent memory / eval 的工程关系
must_avoid:
  - 只翻译摘要
  - 只写“持续学习很重要”但不给工程转化
evidence_required:
  - 论文摘要或 taxonomy 内容
  - 至少 3 个关键概念解释
human_decision: adopt
failure_tags: []
notes: 这是 Self-Improving Research Agent 理论底座之一。
```

### GS-002：自动化科研 Survey

```yaml
id: GS-002
source_type: paper
input: https://victorchen96.github.io/auto_research_survey.pdf
expected_output_type: deep_dive
must_include:
  - L1-L5 autonomy taxonomy
  - 四类 architecture pattern
  - cognitive loop / context limitation / novelty eval / reproducibility / safety / cost
  - 和作品集项目的关系
must_avoid:
  - 把自动化科研等同于自动写论文
  - 忽略 evaluation 和 human-in-the-loop
evidence_required:
  - 论文摘要或 autonomy level 表
  - 至少 3 个失败模式
human_decision: adopt
failure_tags: []
notes: 这是 Self-Improving Research Agent 架构底座之一。
```

### GS-003：GitHub Agent 项目调研

```yaml
id: GS-003
source_type: github
input: 任意一个 AI Agent 开源项目链接
expected_output_type: daily_brief
must_include:
  - 项目解决什么问题
  - 核心架构或核心代码路径
  - 适合学习 / 求职 / 内容哪一类价值
  - 明确是否值得 deep dive
must_avoid:
  - 只列 stars 和 README slogan
  - 不看核心目录就下结论
evidence_required:
  - README 关键段落
  - repo 目录或核心文件
  - 最近更新时间或维护信号
human_decision: revise
failure_tags:
  - weak_evidence
notes: 后续用真实项目填充。
```

### GS-004：日报候选筛选

```yaml
id: GS-004
source_type: news
input: 一组 10 条 AI 新闻 / 项目候选
expected_output_type: daily_brief
must_include:
  - 只选 3-5 条最有价值内容
  - 每条说明为什么值得读
  - 给出对老板学习 / 求职 / 内容线的动作建议
must_avoid:
  - 平铺所有新闻
  - 没有取舍标准
  - 只做行业判断没有动作
evidence_required:
  - 每条候选至少一个来源链接
human_decision: revise
failure_tags:
  - too_generic
notes: 对齐老板偏好：内容要利他、有实用动作。
```

### GS-005：X 草稿生成

```yaml
id: GS-005
source_type: internal_note
input: 一篇 AI Agent 学习笔记
expected_output_type: x_draft
must_include:
  - 一个窄切入点
  - 对读者有用的判断框架或具体动作
  - 不自动发布，只输出草稿
must_avoid:
  - 过度宏大叙事
  - 空泛行业判断
  - 未授权自动发布
evidence_required:
  - 对应笔记链接或原文依据
human_decision: revise
failure_tags:
  - not_actionable
notes: 遵守 X 维护策略：只读、草稿、建议；发布需浏览器链路和明确授权。
```
